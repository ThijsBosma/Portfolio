Project description

In this project we got tasked with making any type of game we wanted that had to be compatible with WebGL and posted to itch.io after 8 weeks. For this project I got chosen for a exchange project which still had the same task but I would be working with students from Finland.

When we heard we were able to make whatever we wanted we brainstormed for quite a while and decided to go with a puzzle game. We really wanted to build a puzzle game because of the infinite amount of possibilities and challenges it could come with. Our game was based a lot on captain toad treasure tracker and we decided to use the same type of camera movement you see in that game.

Seeing as I got chosen for the exchange project this came with multiple extra difficulties the biggest one being that for 6 weeks we had to work abroad and not be able to communicate with everyone face to face. On discord we often let everyone know what we were doing and if certain things were doable or not. This taught me alot for communicating with people when im not right next to them and be able to help other people even though they are in a entire different country then I am.

My additions

Moving platform

Summary

We wanted a quick way to get around the stage and add verticality without using stairs

In our game we decided to add moving platforms that can take certain pickubable objects like a watering can and get them from point A to B, this also included the player. To activate these moving platforms there were multiple things located throughout the level like levers and buttons that when used would activate certain platforms.

When the platform would reach its end we also had to make sure it would be able to loop back so that the player would not get stuck on a certain point of the level.

Player

Summary

I worked on a few things for the player such as climbing, model swap and animations

I worked on a few things for the player aswell, I gave him functionality such as being able to climb, model swap, animations and give it an entire different script where we could save all the data so we would have been able to save it as a JSON file.

We wanted to implement the climbing to be able to give elevation to the levels without having to use moving platforms everywhere, we showed climbable spots by placing down vines where the player can climb up. In level 2 we forced the player to climb up a vine and this way they learned that vines are climbable and help you reach certain spots.

When we got the player model there were quite a few things I had to change, this included: Changing certain colliders so they matched up with the new model, pickup and hold positions for when your picking items up and most importantly add the animations so it looked alot less stale. For the animations I used the animator in unity with certain parameters. I had a playerAnimator script in which I changed the animator parameter based on if certain actions were happening for example the player moving.

We wanted to be able to save the progress of the player so they did not have to constantly restart from scratch. We wanted to save the PlayerData to a JSON file and then load that file again when the player launched the game on itch so they still had their progress. Sadly because of the time we were unable to make this work on itch but we did have a working variant for if we ever released it on desktop

Project description

In this project we were tasked to make a racing game with a night theme. The goal in this game is to race around the track and get the best time possible. The way we motivate you to do this is by having multiple characters that you need a specific time to unlock.

My additions

Ghost system

In the game we wanted to show the player what his best time was, the way we decided to implement this is with a ghost of the player. This system tracks the movements of the player and saves this if its the best time you have driven and replays the saved movements the next race

Learn more

Checkpoints

In the game we decided to place down checkpoints on certain spots on the track so you could easily respawn at that checkpoint and figure out the best way to go about a certain section. If you got stuck somewhere on the map you could respawn without having to reset the race.

Learn more

Game feel

The game itself was quite simple but we still needed to give the feeling of speed to the player without having the car go extremely fast. The way we did this was by adding camera shake and a dynamic camera that goes further back when you speed up and goes more to the front when you are braking.

Project description

For this project our group got tasked with making a game, we could make anything we wanted to make and our group decided on making a roguelike.

The reason we wanted a roguelike was because it would come with multiple challenges and we all could learn a lot from this genre

The goal of the game is to survive the chambers and kill all the skeletons in order to proceed onwards and beat the level

My additions

Enemies

I had the task of making the enemies work in the game. For this I decided to use a brain script that was given to us at the start of the project and added my own things when necessary. The enemies did not have too many behaviours they had to be able to wander and chase the player to attack

I wanted to make the system really easy to work with for other people so I decided that I would make a base enemy script in order to use inheritance.

Learn more

Enemy variants

Because we wanted variety with the enemies the player was facing I also made some extra but simple variants of enemies using the base script. These included a fast enemy that moves quickly but has less health and a tank enemy that had more health but moved a bit slower.

Learn more

Animations

We decided to use animations for the weapons of the player. This way you can easily add new weapons and weapon types to the game because you only had to make the animation for it. To perform certain actions of the weapon we used animation events to put an hit box on for example.

For the skeleton I put the model in mixamo and used those animations to make the skeleton walk around and shoot his bow. I also used animation events to make the skeleton perform certain actions at specific keyframes.

Particle effects

There were a few simple particle effects that I decided to make to improve the game feel. These were the dash effect, effect when hitting an enemy, and a smoke effect that comes of from the torches
