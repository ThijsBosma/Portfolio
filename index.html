Project description

In this project we were tasked to make a racing game with a night theme. The goal in this game is to race around the track and get the best time possible. The way we motivate you to do this is by having multiple characters that you need a specific time to unlock.

We decided not too have many obstacles because we wanted to put more of an emphasis on finding the good racing lines and getting a good time, similar to something like trackmania

My additions

Ghost system

For the game wanted to have a way to easily see how quickly you went through the track, the way we wanted to show this was through the use of a ghost system. This system tracks the movements of the player and saves this if its the best time you have driven and replays the saved movements the next race

Learn more

Checkpoints

In the game we decided to place down checkpoints on certain spots on the track so you could easily respawn at that checkpoint and figure out the best way to go about a certain section. Or if you got stuck somewhere on the map you could respawn without having to reset the race.

Learn more

Game feel

The game itself was quite simple but we still needed to give the feeling of speed to the player without having the car go extremely fast. The way we did this was by adding camera shake a dynamic camera that goes further back when you speed up and goes more to the front when you are braking.

Project description

For this project our group got tasked with making a game, we could make anything we wanted to make and our group decided on making a roguelike.

The goal of the game is to survive the chambers and kill all the skeletons in order to proceed onwards and beat the level

My additions

Enemies

I had the task of making the enemies work in the game. For this i decided to use a brain script that was given to us at the start of the project and added my own things when necessary. The enemies did not have too many behaviours they had to be able to wander and chase the player to attack

I wanted to make the system really easy to work with for other people so, i decided that i would make a base enemy script in order to use inheritance.

Learn more

Enemy variants

Because we wanted variety with the enemies the player was facing i also made some extra but simple variants of enemies using the base script. These included a fast enemy that moves quickly but has less hp and a tank enemy that had more hp but moved a bit slower

Learn more

Animations

We decided to use animations for the weapons of the player, this way you can easily add new weapons and weapon types to the game because you only had to make the animation for it. To perform certain actions of the weapon we used animation events to put a hit box on for example.

For the skeleton i put the model in mixamo and used those animations to make the skeleton walk around and shoot his bow. I also used animation events to make the skeleton perform certain actions at specific keyframes.

Particle effects

There were a few simple particle effects that i decided to make to improve the game feel. These were the dash effect, effect when hitting a enemy, and an effect that comes of from the torches

Project description

For this project we were tasked with making a twinstick shooter in 8 weeks. This was a group project consisting of 2 developers and 3 artists

The goal of the game is to walk around the facility and try and find the necessary keycard to continue in the level. You have to fight your way through droids that have been taken over by a virus. Its your task to get to the end and reset all the robots

My additions

Keycard system

Because you had to go through the facility as fast as possible we still wanted to give the player some difficultys so he could not get to the end immediately. The way we did this was by adding keycards, we would place these all across the map and you would have to find the correct one to get to the next level

Learn more

Health system

In the game you had both a healthbar and shieldbar, i decided to implement this with inheritance so you can easily make a new script for the player or enemy health and easily get the required things for health

Learn more

Dash

We needed to give the player a way to dash and weave away from enemys when being shot at so you have a better chance of surviving. Thats the reason why we decided to implement a dash, the dash has a cooldown and will shoot you in any direction you are currently holding

Learn more

Drop system enemy

We wanted to make it so the player would have to kill all the enemys in a certain room in order to progress, the way we decided to implement this is by making a system where if all enemys are dead the final one will drop a keycard that is required to get to the next level

Project description

In this project i got tasked with making a game where the main mechanic was that there had to be a marble that rolled around. I decided to take this idea and make a third person platform.

The objective is to get to the end of the level. One level you can easily go to the end by following a straightforward path and the other you need to collect gold bars to get to the next level

My additions

Movement

For the game i had to create third person movement in order to move around, the thing i had to keep in mind though is that the camera is constantly changing position and that when holding forward you should always go to where the camera is facing

Learn more

Collecting system

I wanted to make a big level at the end where you had to collect multiple golden bars in order to unlock the next level. The way i implemented this feature in the final level is by making 3 different sectionsm, one that was easy one that was medium difficulty and one that was hard.

Learn more

Controller support

Usually platformers play alot nicer with controller then on keyboard so i decided to add controller support to my game. I did this with the current input system from unity through the input manager.
